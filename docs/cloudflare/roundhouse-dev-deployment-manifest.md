## Authentication

Access protects the entire hostname. The human policy uses decision `allow` and
an exact email selector. The smoke policy uses decision `non_identity` and the
exact generated service-token ID; it does not accept every account service
token. No bypass policy is created.

The Worker independently validates `Cf-Access-Jwt-Assertion` using the Access
JWKS endpoint. It requires:

- RS256 signature accepted by `jose`;
- issuer `https://<approved-team>.cloudflareaccess.com`;
- the exact application AUD returned by Access creation;
- token type `app`;
- an `email` or service-token `common_name` actor.

The only Worker egress introduced is JWKS retrieval from the exact approved
Access team hostname. Keys are cached by `jose`. The service-token client secret
is shown once by Cloudflare, stored only in a mode-0600 local file outside the
repository, used as request headers, and deleted with that file after smoke.

The required scoped API credential needs only the applicable account Access Apps
and Policies Read/Write and Access Service Tokens Read/Write permissions plus
zone DNS and SSL read access for collision/audit checks. Existing Wrangler OAuth
is used for approved Worker, D1, Queue, and Custom Domain operations. No new API
token is created by automation.
